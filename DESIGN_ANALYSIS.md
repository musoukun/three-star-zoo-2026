# Three Star Zoo 設計思想分析

## 概要

本プロジェクトは Boardzilla フレームワーク上に構築されたボードゲーム「三ツ星動物園」の実装である。
**動物の追加を容易にする拡張性の高い設計**を目指しているが、核心部分である**効果処理エンジンが未実装**のまま中途半端な状態にある。

---

## 1. 動物追加を容易にする設計

### ファクトリ/レジストリパターン

動物は個別ファイル（プレーンオブジェクト）として定義し、`Animal/index.ts` の `animalMap` に登録するだけで追加できる設計になっている。

```
src/Animal/
├── index.ts              ← animalMap（レジストリ）
├── AnimalColor.ts        ← 色の定義
├── RessaPanda.ts         ← 個別の動物定義
├── Penguin.ts
├── Lion.ts
└── ...（計11種）
```

**レジストリ（`Animal/index.ts`）:**
```typescript
export const animalMap = {
  RessaPanda,
  Penguin,
  AfricanElephant,
  // ← ここに追加するだけで新動物がゲームに参加
};
```

**個別動物の定義例（`RessaPanda.ts`）:**
```typescript
export const RessaPanda: any = {
  id: "RessaPanda",
  name: "レッサーパンダ",
  cost: 0,
  poops: 0,
  color: [AnimalColor.RED, AnimalColor.GREEN, AnimalColor.ORANGE],
  effect: {
    global: false,
    timing: "first",
    creation: 0,
    buff: [1, "RessaPanda", "each"],
    bonusbuff: [1, "GiantPanda", "once"],
  },
  inventory: 8,
  global: false,
};
```

**設計意図:** クラスを書く必要すらなく、決まった形式のオブジェクトを定義してマップに登録するだけで動物を追加できる。ゲームロジック本体を触る必要がない。

## 2. 効果システムの設計（意図されていた姿）

### AnimalEffect インターフェース

`game/index.ts` に定義されたインターフェースから、以下のような効果タイプを想定していたことがわかる：

```typescript
export interface AnimalEffect {
  global?: boolean;           // 他プレイヤーのダイスでも発動するか
  timing?: "first" | "end";  // 発動タイミング（先発 or 後発）
  creation?: number;          // 固定コイン生成量
  creationIf?: [string, string, string, string, string, string, string];
                              // 条件付きコイン生成（独自DSL）
  buff?: [number, string, "each" | "once"];
                              // 特定動物の数に応じたバフ
  bonusbuff?: [number, string, "each" | "once"];
                              // ボーナスバフ
  steal?: number;             // 固定コイン窃盗量
  stealIf?: [string, string, string, string, string, string, string];
                              // 条件付き窃盗（独自DSL）
  choice?: AnimalEffect[];    // プレイヤーが効果を選択
}
```

### 条件式DSL（独自ミニ言語）

`creationIf` / `stealIf` に三項演算子風の独自DSLが使われている：

```typescript
// ペンギンの例: ペンギンが2匹以上なら2コイン、そうでなければ1コイン
creationIf: ["Penguin", ">=", "2", "?", "2", ":", "1"]
//            対象動物   比較演算子  閾値  ?   真の値  :  偽の値

// ライオンの例: ライオンが2匹以上なら2コイン窃盗
stealIf: ["Lion", ">=", "2", "?", "2", ":", "1"]
```

### 効果タイプの分類（全動物から読み取れるパターン）

| 効果パターン | 対応フィールド | 使用動物 |
|---|---|---|
| 固定コイン生成 | `creation: N` | チーター(3), アフリカゾウ(5), コザクラインコ(1) |
| 条件付きコイン生成 | `creationIf: [DSL]` | ペンギン |
| バフ（動物数連動） | `buff: [N, "動物名", "each"/"once"]` | レッサーパンダ, ジャイアントパンダ |
| ボーナスバフ | `bonusbuff: [N, "動物名", "each"/"once"]` | レッサーパンダ |
| 固定窃盗 | `steal: N` | カリフォルニアアシカ |
| 条件付き窃盗 | `stealIf: [DSL]` | ライオン |
| 選択式効果 | `choice: [効果A, 効果B]` | ハンドウイルカ |
| スター窃盗 | `steal: [N, "target", 1]` | シロサイ |
| 隣接ケージ効果 | `adjacent` フィールド | アミメキリン |

### 想定されていた処理フロー

```
ダイスロール
  ↓
該当ケージ番号の動物を取得
  ↓
各動物の effect を評価
  ├── timing が "first" → 先に処理
  ├── timing が "end" → 後で処理
  ├── global が true → 他プレイヤーのダイスでも発動
  ↓
効果タイプ別に分岐
  ├── creation → コイン付与
  ├── creationIf → DSL評価 → 条件分岐でコイン付与
  ├── buff → 場の動物数カウント → コイン付与
  ├── steal → 対象プレイヤー選択 → コイン窃盗
  ├── stealIf → DSL評価 → 条件分岐で窃盗
  ├── choice → プレイヤーに選択UI表示 → 選んだ効果を実行
  └── adjacent → 隣接ケージの動物もカウント
```

## 3. ゲーム全体のアーキテクチャ

### ターン進行FSM（ステートマシン）

```
POOP → ROLL → INCOME → TRADE → FLUSH → (次のプレイヤーの POOP)
 │       │       │        │       │
 │       │       │        │       └─ うんち清掃・ターン終了
 │       │       │        └─ 動物の売買・スター購入
 │       │       └─ 効果による収入処理 ← ★ここが未実装
 │       └─ ダイスロール
 └─ うんちコスト支払い
```

### クラス構成

```
Game (Boardzilla)
  └── ThreeStarZooGame        ゲーム状態管理・FSM
        ├── ThreeStarZooPlayer  プレイヤー（coins, stars, inventory）
        ├── Cage (Space)        ケージ（1-12番、最大2匹）
        │     └── Animal (Piece)  動物ピース
        └── Board Layout
              ├── market-area     マーケット
              ├── player-cages-area  プレイヤーのケージ群
              ├── controls-area   操作パネル
              └── other-players-area 他プレイヤー表示
```

### 勝利条件

- スター3つで勝利（`stars >= 3`）
- スターは10コインで購入

## 5. 設計思想のまとめ

### 目指していた方向性

1. **データ駆動設計** - 動物をコードではなくデータ（オブジェクトリテラル）で定義し、効果もデータとして記述する。新しい動物の追加にゲームロジックの変更が不要。

2. **宣言的な効果定義** - 効果をプログラムではなく「何が起きるか」の宣言として記述。`creation: 3` は「3コイン生成」、`buff: [1, "RessaPanda", "each"]` は「レッサーパンダ1匹につき1コイン」という宣言。

3. **独自DSLによる条件分岐** - `creationIf` / `stealIf` の7要素タプルで簡易的な条件式を表現。パーサー/インタプリタを書けばどんな条件でも対応できる拡張ポイント。

4. **ファクトリパターンによる疎結合** - `animalMap` に登録するだけで動物がゲームに参加。個々の動物ファイルは互いに依存しない。

### 完成に必要な実装

効果処理エンジン（`triggerCageEffects` の中身）として、以下が必要：

1. **DSLパーサー** - `creationIf` / `stealIf` の7要素タプルを評価する関数
2. **バフ計算器** - 場の動物数をカウントして `buff` / `bonusbuff` を計算
3. **タイミング制御** - `"first"` / `"end"` で処理順を分ける
4. **グローバル判定** - `global: true` の動物は全プレイヤーのダイスで発動
5. **選択UI連携** - `choice` フィールドの効果をプレイヤーに選ばせるUI
6. **対象選択** - `steal` 系の効果で窃盗対象プレイヤーを選択するUI
7. **隣接ケージ参照** - アミメキリンの `adjacent` による隣接ケージ参照
